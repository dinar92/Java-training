    Переопределенный метод equals() должен соответствовать 
следующим правилам:
  1) Рефлексивность. для не null x: x.equals(x) должен быть true.
  2) Транзитивность. Для не null x, y, z: Если x.equals(y) == true, и y.equals(z) == true, то и x.equals(z) должен быть true.
  3) Симметричность. Для не null x, y: Если x.equals(y) == true, то 
 и y.equals(x) должен быть true.
  4) Постоянство. При любом количестве вызовов x.equals(y) 
должен возввращать одинаковый результат при если свойства объектов не изменялись.
  5) Не равенство null. Для не null x: x.equals(null) должен 
всегда возвращать false.
    При переопределении equals() следует всегда переопределять hashCode() с целью корректной работы 
    с коллекциями на базе хеш-таблиц.
